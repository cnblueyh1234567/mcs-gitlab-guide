💡 Git คืออะไร? — อธิบายแบบเด็ก 5 ขวบเข้าใจ

ลองนึกภาพว่าเรากำลังเขียนรายงานกลุ่มกับเพื่อน 5 คน — ถ้าทุกคนแก้ไฟล์เดียวกันพร้อมกันจะปนกันหมด!

Git = ระบบจัดการ version ที่ให้ทุกคนแก้ "สำเนาของตัวเอง" แล้วรวมกันทีหลังได้

GitLab = เว็บที่เก็บ code ของทีม (เหมือน Google Drive สำหรับ code)

🧩 4 คำที่ต้องรู้ (คลิกเพื่อดูรายละเอียด)

🌳
Branch
เปรียบเหมือน: สมุดทด
📓
main = สมุดจริงที่ส่งอาจารย์
branch = สมุดทดที่ลองเขียนก่อน
ถ้าเขียนดี → ลอกใส่สมุดจริง
ถ้าพัง → ฉีกทิ้ง สมุดจริงยังปลอดภัย!
# สร้าง "สมุดทด" ใหม่
git checkout -b fix/export-button
💾
Commit
เปรียบเหมือน: กด Save Game
🎮
ทุกครั้งที่ผ่าน checkpoint เรากด save
ถ้าตาย → โหลด save กลับมาได้!
ยิ่ง save บ่อย ยิ่งปลอดภัย
# เพิ่มไฟล์ + กด save
git add .
git commit -m "fix: แก้ปุ่ม Export"
🚀
Push
เปรียบเหมือน: อัปโหลดขึ้น Cloud
☁️
commit = save ในเครื่อง (offline)
push = อัปโหลดขึ้น GitLab (online)
ถ้าไม่ push → คนอื่นจะไม่เห็นงานเรา!
# ส่งงานขึ้น GitLab
git push -u origin fix/export-button
🔀
Merge Request
เปรียบเหมือน: ส่งงานให้หัวหน้าตรวจ
📋
ทำงานเสร็จ → สร้าง MR → ให้คนรีวิว
รีวิวผ่าน → merge เข้า main
MR = Pull Request (PR) ใน GitHub
🎯
หลัง push → GitLab จะแสดง link สำหรับสร้าง MR ให้อัตโนมัติ!

🚦 กฎทอง 3 ข้อ

🚫
ห้ามแก้ code ใน main โดยตรง!
ต้องสร้าง branch ใหม่เสมอ
✅
เขียน commit message ให้ชัด!
เช่น fix: แก้ปุ่ม Export ไม่ทำงาน
⚠️
git pull ก่อนเริ่มงานทุกครั้ง!
ดึง code ล่าสุดเพื่อไม่ให้ชนกับคนอื่น

⚡ คำสั่งที่ใช้บ่อย

🟢 เริ่มงาน

git pull origin main
ดึง code ล่าสุดจาก main
ใช้เมื่อ: ก่อนเริ่มงานทุกครั้ง
git checkout -b ชื่อ-branch
สร้าง branch ใหม่ + สลับไปทันที
ใช้เมื่อ: เริ่มงานใหม่
git checkout ชื่อ-branch
สลับไป branch ที่มีอยู่แล้ว
ใช้เมื่อ: ไปทำงาน branch อื่น
git branch
ดู branch ทั้งหมด (* = อยู่ตรงนี้)
ใช้เมื่อ: งง ว่าอยู่ branch ไหน

🟡 ระหว่างทำงาน

git status
ดูว่าแก้ไฟล์อะไรไปบ้าง
ใช้เมื่อ: ก่อน commit ทุกครั้ง
git diff
ดูรายละเอียดที่เปลี่ยน (บรรทัดต่อบรรทัด)
ใช้เมื่อ: อยากดูว่าแก้ตรงไหน
git add .
เพิ่มทุกไฟล์ที่แก้เข้า staging
ใช้เมื่อ: เตรียม commit
git commit -m "ข้อความ"
บันทึกการเปลี่ยนแปลง
ใช้เมื่อ: แก้ code เสร็จ

🔵 ส่งงาน

git push -u origin ชื่อ-branch
push ครั้งแรก (set upstream)
ใช้เมื่อ: push branch ใหม่ครั้งแรก
git push
push ครั้งต่อไป (ไม่ต้อง -u)
ใช้เมื่อ: push เพิ่มหลังจากครั้งแรก

🔴 แก้ปัญหา

git stash
เก็บงานชั่วคราว (ยังไม่อยาก commit)
ใช้เมื่อ: ต้องสลับ branch แต่งานยังไม่เสร็จ
git stash pop
เอางานที่เก็บกลับมา
ใช้เมื่อ: กลับมาทำต่อ
git log --oneline -5
ดู 5 commits ล่าสุด
ใช้เมื่อ: ตรวจสอบ history
git reset HEAD~1
ยกเลิก commit ล่าสุด (ไฟล์ยังอยู่)
ใช้เมื่อ: commit ผิด อยาก commit ใหม่

❓ คำถามที่พบบ่อย

🤔 commit กับ push ต่างกันยังไง?▼
💾
Commit
Save ในเครื่อง
คนอื่นไม่เห็น
🚀
Push
อัปโหลดขึ้น GitLab
คนอื่นเห็น
ต้อง commit ก่อน แล้วค่อย push — commit ได้หลายครั้งแล้ว push ทีเดียว
😱 commit ผิด branch ทำไงดี?▼
ยังไม่ push:
# ยกเลิก commit ล่าสุด (ไฟล์ยังอยู่)
git reset HEAD~1
# สลับไป branch ที่ถูกต้อง
git checkout branch-ที่ถูกต้อง
# commit ใหม่
git add . && git commit -m "ข้อความ"
push ไปแล้ว: แจ้งพี่เต้ยช่วย! อย่าแก้เอง
🔴 push แล้ว error "rejected" คืออะไร?▼
แปลว่า branch บน GitLab มีอะไรใหม่กว่าในเครื่อง:
# ดึง code ล่าสุดของ branch นี้
git pull origin ชื่อ-branch
# แล้ว push ใหม่
git push
ถ้ามี conflict → แจ้งพี่เต้ย!
💥 merge conflict คืออะไร? แก้ยังไง?▼
เกิดเมื่อ 2 คนแก้ไฟล์เดียวกันตรงบรรทัดเดียวกัน:
<<<<<<< HEAD (ของเรา)
showLoading(true);
=======
showSpinner();
>>>>>>> main (ของคนอื่น)
วิธีแก้: เลือกว่าจะเอา code ส่วนไหน → ลบเครื่องหมาย <<< === >>> → commit ใหม่
💡
ไม่มั่นใจ? แจ้งพี่เต้ยช่วยดู conflict ให้!

🆘 ทำพัง อยากย้อนกลับทำไงดี?▼
# ย้อนไฟล์เดียว (เหมือนก่อนแก้)
git checkout -- ชื่อไฟล์
# ย้อน commit ล่าสุด (ไฟล์ยังอยู่)
git reset HEAD~1
# เริ่มใหม่หมด — กลับ main
git checkout main
🚫
ห้ามใช้ --force หรือ --hard โดยไม่ปรึกษาพี่เต้ยก่อน!
