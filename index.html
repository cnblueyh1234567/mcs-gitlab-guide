💡 Git คืออะไร? ทำไมต้องใช้?

ลองนึกภาพว่าทีม 5 คนแก้ไฟล์เดียวกันพร้อมกัน — ถ้าไม่มี Git จะปนกันหมด! Git ช่วยให้ทุกคนทำงานแยกกัน แล้วรวมกันทีหลังได้อย่างปลอดภัย

🌳
Branch
สำเนาของ code
เปรียบเหมือน: สมุดทดลอง

เวลาจะแก้ code เราไม่แก้ใน main โดยตรง แต่สร้าง "กิ่ง" ออกมาเพื่อทดลองแก้ ถ้าพังก็ลบกิ่งทิ้ง main ยังปลอดภัย

main = code ที่ใช้จริง (production)
feature/xxx = กิ่งที่เราสร้างเพื่อทำงาน
💾
Commit
บันทึกจุดเซฟ
เปรียบเหมือน: กด Save Game

ทุกครั้งที่แก้ code เสร็จ เรา "commit" เพื่อบันทึกไว้ พร้อมเขียนข้อความว่าแก้อะไร ถ้าพลาดสามารถย้อนกลับมาจุดนี้ได้

สำคัญ: เขียน commit message ให้ชัดเจน!
🚀
Push
อัปโหลดขึ้น GitLab
เปรียบเหมือน: อัปโหลดงานขึ้น Cloud

Commit = บันทึกในเครื่องตัวเอง
Push = ส่งขึ้น GitLab ให้คนอื่นเห็น

ถ้าไม่ push คนอื่นจะไม่เห็นงานเรา!
🔀
Merge Request
ขอรวม code
เปรียบเหมือน: ส่งงานให้หัวหน้าตรวจ

เมื่อทำงานเสร็จ สร้าง Merge Request (MR) เพื่อ:
1. ให้คนอื่นรีวิว code
2. ตรวจสอบว่าไม่มีข้อผิดพลาด
3. รวมเข้า main อย่างปลอดภัย

MR = Pull Request (PR) ใน GitHub

🎯 กฎทอง 3 ข้อ

🚫
ห้าม! แก้ code ใน branch main โดยตรง — ต้องสร้าง branch ใหม่เสมอ
✅

⚠️
ระวัง! ก่อนเริ่มงานทุกครั้ง git pull ดึง code ล่าสุดก่อนเสมอ

🗺️ ตัวอย่างสถานการณ์จริง

📋
โจทย์: ได้รับ ticket "แก้ bug ปุ่ม Export ไม่ทำงาน"

ขั้นตอน:
1. git pull origin main — ดึง code ล่าสุด
2. git checkout -b fix/export-button-broken — สร้าง branch
3. แก้ code... ทดสอบ...
4. git add . — เตรียมไฟล์
5. git commit -m "fix: แก้ปุ่ม Export ไม่ทำงาน" — บันทึก
6. git push origin fix/export-button-broken — อัปโหลด
7. เปิด GitLab → สร้าง Merge Request
8. รอรีวิว → Merge! ✅

🟢 ก่อนเริ่มงาน (ทำทุกครั้ง!)

# ดึง code ล่าสุดจาก main
git checkout main
git pull origin main
# สร้าง branch ใหม่
git checkout -b fix/export-button-broken
Switched to a new branch 'fix/export-button-broken'

🟡 ระหว่างทำงาน

# ดูว่าแก้ไฟล์อะไรไปบ้าง

# เพิ่มไฟล์ที่จะ commit (ทุกไฟล์)
git add .
# commit พร้อมข้อความ
git commit -m "fix: แก้ปุ่ม Export ไม่ทำงาน"

📋 Cheatsheet คำสั่งที่ใช้บ่อย

คำสั่ง ทำอะไร ใช้เมื่อ
git status ดูสถานะไฟล์ อยากรู้ว่าแก้ไฟล์อะไรไป
git pull ดึง code ล่าสุด ก่อนเริ่มงานทุกครั้ง
git checkout -b ชื่อ สร้าง branch ใหม่ เริ่มงานใหม่
git checkout ชื่อ สลับ branch ไปทำงาน branch อื่น
git add . เพิ่มทุกไฟล์เข้า staging เตรียม commit
git commit -m "ข้อความ" บันทึกการเปลี่ยนแปลง แก้ code เสร็จ
git push อัปโหลดขึ้น GitLab ส่งงานให้คนอื่นเห็น
git log --oneline -5 ดู commits ล่าสุด ตรวจสอบ history
git diff ดูว่าเปลี่ยนอะไรไป ก่อน commit
git stash เก็บงานชั่วคราว ต้องสลับ branch แต่ยังไม่อยาก commit

❓ คำถามที่พบบ่อย

🤔 commit กับ push ต่างกันยังไง? ▼
Commit = บันทึกในเครื่องตัวเอง (เหมือนกด Save)
Push = ส่งขึ้น GitLab (เหมือนอัปโหลดขึ้น Cloud)

ต้องทำ commit ก่อน แล้วค่อย push เสมอ สามารถ commit หลายครั้งแล้ว push ทีเดียวได้
😱 commit ผิด branch ทำไงดี? ▼
ถ้ายังไม่ push:
git reset HEAD~1 — ยกเลิก commit ล่าสุด (ไฟล์ยังอยู่)
แล้วสลับไป branch ที่ถูกต้อง แล้ว commit ใหม่

ถ้า push ไปแล้ว:
แจ้งพี่เต้ย! อย่าแก้เอง เดี๋ยวพี่ช่วย
🔴 push แล้วขึ้น error "rejected" คืออะไร? ▼
แปลว่า branch บน GitLab มีอะไรใหม่กว่าในเครื่อง ให้ทำ:

git pull origin ชื่อ-branch

ถ้ามี conflict (ไฟล์ชนกัน) — ดูไฟล์ที่มีเครื่องหมาย <<<< แก้ให้เรียบร้อย แล้ว commit + push ใหม่

ถ้าไม่มั่นใจ แจ้งพี่เต้ย!
🌳 ตั้งชื่อ branch ยังไงดี? ▼
ใช้ prefix ตามประเภทงาน:

fix/ชื่อ-bug — แก้ bug
feat/ชื่อ-feature — เพิ่มฟีเจอร์ใหม่
test/ชื่อ-test — เพิ่ม test
refactor/ชื่อ-งาน — ปรับโครงสร้าง code

ตัวอย่าง: fix/ticket-export-button, feat/ticket-mr-creation
📝 Merge Request ต้องเขียนอะไรบ้าง? ▼
Title: สั้น ชัด ตรงประเด็น
เช่น "fix: แก้ปุ่ม Export ไม่ทำงาน"

Description:
1. สิ่งที่แก้ไข — บอกว่าแก้อะไรบ้าง
2. วิธีทดสอบ — บอกว่าต้องทดสอบยังไง
3. Screenshot — ถ้ามีหน้า UI เปลี่ยน ถ่ายรูปแนบ

Assign: เลือกคนรีวิว (ปกติเลือกพี่เต้ย)
💥 merge conflict คืออะไร? แก้ยังไง? ▼
เกิดเมื่อ 2 คนแก้ไฟล์เดียวกันตรงบรรทัดเดียวกัน Git ไม่รู้ว่าจะเอาของใคร

วิธีแก้:
1. เปิดไฟล์ที่ conflict
2. หาเครื่องหมาย <<<<<<< และ >>>>>>>
3. เลือกว่าจะเอา code ส่วนไหน (หรือรวมทั้งสอง)
4. ลบเครื่องหมาย conflict ออก
5. git add . แล้ว git commit

💡
ถ้าไม่มั่นใจ อย่าแก้เอง! แจ้งพี่เต้ยช่วยดู conflict ให้
🔄 git pull กับ git fetch ต่างกันยังไง? ▼
git fetch = ดึงข้อมูลมาดู แต่ไม่แก้ไฟล์ในเครื่อง
git pull = fetch + merge (ดึงมาและรวมเข้า code เลย)

สำหรับมือใหม่: ใช้ git pull เลย ง่ายกว่า!
🆘 ทำพัง อยากย้อนกลับทำไงดี? ▼
ย้อนไฟล์เดียว:
git checkout -- ชื่อไฟล์

ย้อน commit ล่าสุด (ไฟล์ยังอยู่):
git reset HEAD~1

ย้อนทุกอย่างเหมือน main:
git checkout main แล้วสร้าง branch ใหม่

🚫
ห้ามใช้ git reset --hard หรือ --force โดยไม่ปรึกษาพี่เต้ยก่อน!
